# Common Design Mistakes

실전에서 자주 발생하는 실수 패턴과 교정 방법.

## 에셋/스프라이트 관련

| 실수 | 원인 | 교정 |
|------|------|------|
| 스프라이트 실물 안 보고 이름으로 추측하여 적용 | "BtnDefault니까 버튼 배경이겠지" | 반드시 프리뷰로 형태/색상/내장 요소 확인 후 적용 |
| color 틴트로 스프라이트 톤 억지 맞춤 | 원래 시안인 스프라이트에 흰색 틴트 → 시안 폭발 | 톤이 맞는 에셋을 고르거나 단색으로 대체 |
| 9-slice border 없는 스프라이트를 Sliced로 사용 | border=(0,0,0,0)이면 단순 스트레치와 동일 | border 확인 후 적용. 없으면 Simple 모드 또는 직접 border 설정 |
| "완성된 디자인" 스프라이트 위에 추가 요소 겹침 | BtnBack(화살표 내장) 위에 텍스트 → 시각 충돌 | 스프라이트 내장 요소를 활용하거나, 빈 프레임 스프라이트를 쓸 것 |
| 여러 에셋 팩 톤 혼용 | Wenrexa(네온 Sci-Fi) + Heat(미니멀 플랫) 혼합 | 한 팩의 범용 요소만 취사선택. 완성형 스프라이트는 해당 팩 전용 |

## 비례/크기 관련

| 실수 | 원인 | 교정 |
|------|------|------|
| 요소 크기에 비해 테두리가 과도하게 두꺼움 | 1024px 텍스처의 25px border를 400px 카드에 적용 | border 값 / 요소 크기 비율 확인. 1% 이하가 적절 |
| 패딩 없이 텍스트가 경계에 붙음 | 숨 쉴 공간 부재 | 최소 8-12px 패딩 |
| 버튼 크기가 불일치 | 같은 역할의 버튼이 각기 다른 크기 | 동일 역할 = 동일 크기 (Repetition 원칙) |

## 간격 관련

| 실수 | 원인 | 교정 |
|------|------|------|
| 내부 간격 >= 외부 간격 | 카드 내부 24px, 카드 간 16px → 그룹 경계 모호 | 내부 < 외부 규칙 적용 |
| 임의 간격 (13px, 37px) | 그리드 시스템 없음 | 8px 그리드에 스냅 |
| 빈 공간이 "비어 보임" | 의도 없는 여백 | 여백을 줄이거나, 의도적 여백으로 시각적 호흡 구간 형성 |

## 색상/대비 관련

| 실수 | 원인 | 교정 |
|------|------|------|
| 전체에 액센트 색상 과용 | 시안이 배경, 텍스트, 보더, 바 전부에 사용 | 전체 면적의 10-15%로 제한 |
| 배경과 전경 분리 부족 | 카드 배경(0.10)과 화면 배경(0.06)의 차이 4% → 경계 불명확 | 레이어 간 밝기 차이 충분히 확보하거나, 1px 보더로 보조 |
| 다크 배경에 고채도 색상 | 눈이 피로해지는 vibration 효과 | 채도 낮추고 밝기 올려서 부드럽게 |

## 위계/흐름 관련

| 실수 | 원인 | 교정 |
|------|------|------|
| 모든 요소가 비슷한 시각적 무게 | 제목, 본문, 레이블이 같은 크기/굵기 | 크기, 색상, 굵기로 위계 명확히 |
| CTA가 눈에 안 띔 | 주요 버튼과 보조 버튼의 시각적 차이 부족 | CTA = 화면에서 가장 눈에 띄는 요소 |
| 시선 흐름 무시한 배치 | 중요 정보가 우하단, 보조 정보가 좌상단 | Z/F 패턴에 맞게 재배치 |
