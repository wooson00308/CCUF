# Color Theory & Dark UI

## 대비 비율 기준 (WCAG)

- 일반 텍스트: 최소 4.5:1
- 큰 텍스트 (24px+ 또는 19px+ bold): 최소 3:1
- 장식/비필수 요소: 기준 없음, 하지만 3:1 이상 권장
- 흑백 최대 대비: 21:1

## 다크 UI 배경 레이어링

순검정(#000) 대신 근검정을 사용. 각 레이어는 2-4% 밝기 차이로 깊이 표현.

| 레이어 | 용도 | RGB 범위 |
|-------|------|---------|
| Level 0 | 최하위 배경 | (0.06-0.07, 0.07-0.08, 0.09-0.10) |
| Level 1 | 패널/사이드바 | (0.08-0.10, 0.09-0.11, 0.11-0.14) |
| Level 2 | 카드/모달 | (0.12-0.14, 0.13-0.15, 0.16-0.18) |
| Level 3 | 호버/선택 상태 | (0.15-0.18, 0.17-0.20, 0.20-0.25) |
| Level 4 | 팝업/오버레이 | (0.18-0.22, 0.20-0.24, 0.24-0.28) |

핵심: 각 레이어가 "한 단계 떠 보이는" 느낌을 줘야 함.

## 텍스트 대비 계층

| 역할 | 알파/밝기 | 예시 |
|------|---------|------|
| 주요 텍스트 (제목, 본문) | 87-100% 흰색 (#E0E0E0 ~ #F0F0F0) | 파일럿 이름, 스탯 수치 |
| 보조 텍스트 (레이블, 힌트) | 60% 흰색 (#9E9E9E ~ #B0B0B0) | 스탯 이름, 카테고리 |
| 비활성 텍스트 | 38% 흰색 (#616161 ~ #757575) | disabled 버튼 텍스트 |
| 순백 (#FFFFFF) | 제한적 사용 | 최상위 강조 제목에만 |

## 채도 관리

- 다크 배경에서 고채도 색상은 시각적 "vibration" 유발
- 액센트 색상은 채도를 낮추거나 밝기를 올려서 사용
- 좋은 시안 액센트: (0.35, 0.75, 0.85) — 채도 적절, 밝기 충분
- 나쁜 시안: (0.0, 1.0, 1.0) — 채도 과잉, 눈에 거슬림

## 액센트 색상 사용 규칙

- 전체 화면에서 액센트 색상이 차지하는 면적: 10-15% 이하
- 사용처: CTA 버튼, 선택 상태 인디케이터, 중요 수치, 진행 바
- 사용 금지: 배경, 대면적 패널, 일반 텍스트
- 너무 많이 쓰면 "강조"가 "기본"이 되어 위계가 사라짐

## 색상으로 의미 전달 시

- 색상만으로 정보를 전달하지 않음 (색맹 대응)
- 반드시 아이콘, 텍스트, 위치 등 추가 신호와 병행
- 상승 = 초록 + 화살표(Up), 하락 = 빨강 + 화살표(Down)

## color 틴트 적용 시 주의

- 스프라이트에 color를 곱하면 원래 색감이 변질됨
- 어두운 스프라이트에 밝은 color = 의도치 않은 색상 노출
- 밝은 스프라이트에 어두운 color = 디테일 매몰
- 원칙: 스프라이트 원색을 살리려면 color = white. 톤 조절이 필요하면 전용 스프라이트를 쓸 것.
