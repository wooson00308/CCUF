# Spacing & Grid System

## 8px Grid

모든 간격을 8의 배수로 통일하면 시각적 리듬이 생긴다.

- 스케일: 4, 8, 16, 24, 32, 40, 48, 56, 64
- 4px는 미세 조정용 (아이콘-텍스트 간격 등)
- 7, 13, 19 같은 임의 수치는 리듬을 깨뜨림

## 내부 < 외부 규칙

컴포넌트 내부 패딩은 컴포넌트 간 마진보다 작아야 한다. 이것이 게슈탈트 근접성의 실전 적용.

- 카드 내부 패딩 16px → 카드 간 간격 최소 24-32px
- 버튼 내부 패딩 12px → 버튼 간 간격 최소 16px
- 섹션 내부 간격 24px → 섹션 간 간격 최소 48px

이 규칙이 깨지면 "어디서부터 어디까지가 한 그룹인지" 인식이 흐려짐.

## 컴포넌트별 권장 간격

| 컴포넌트 | 내부 패딩 | 요소 간 간격 |
|---------|---------|------------|
| 버튼 | 12-16px 수평, 8-12px 수직 | 버튼 간 8-16px |
| 카드 | 16-24px | 카드 간 16-24px |
| 리스트 아이템 | 12-16px | 아이템 간 4-8px (compact) / 8-12px (normal) |
| 섹션 | 24-32px | 섹션 간 32-48px |
| 화면 가장자리 | 16-24px 마진 | — |
| 헤더/푸터 바 | 16-24px 수평, 8-16px 수직 | 내부 요소 간 16-24px |

## 터치/클릭 타겟

- 최소 터치 영역: 44x44px (모바일), 32x32px (데스크탑)
- 터치 타겟 간 최소 간격: 8px
- 시각적 크기와 히트 영역은 다를 수 있음 — 작은 아이콘도 히트 영역은 44px로

## 타이포그래피 스케일

기본 비율로 일관된 크기 세트를 생성:

- Major Third (1.25): 12 → 16 → 20 → 25 → 32 — 촘촘한 위계, 밀도 높은 UI에 적합
- Perfect Fourth (1.333): 12 → 16 → 21 → 28 → 38 — 웹 UI 표준
- Golden Ratio (1.618): 12 → 16 → 26 → 42 — 극적인 위계, 제한적 사용

프로젝트 내에서 하나의 비율만 사용할 것. 혼용하면 위계가 흐려짐.
