# Design Foundations

## CRAP 원칙 (Robin Williams)

### Contrast (대비)
- 요소 간 차이는 "살짝 다르게"가 아니라 "확실히 다르게". 애매한 차이는 실수처럼 보임.
- 크기, 색상, 굵기, 형태 중 최소 하나는 확실히 달라야 함.
- "비슷하게 만들 거면 똑같이 만들고, 다르게 만들 거면 확실히 다르게."

### Repetition (반복)
- 같은 역할 = 같은 시각 언어. 색상, 폰트, 간격, 컴포넌트 형태를 반복.
- 반복이 깨지면 사용자는 "이건 다른 건가?"라고 혼란.
- 반복이 브랜드 톤을 만든다.

### Alignment (정렬)
- 모든 요소는 보이지 않는 선 위에 놓여야 한다.
- 의도적 정렬 깨짐이 아니면 전부 맞출 것.
- 중앙 정렬은 "약한 정렬" — 좌정렬이 시선 흐름에 더 강함.

### Proximity (근접성)
- 관련 있는 것은 가깝게, 관련 없는 것은 멀게.
- 핵심 규칙: 내부 간격 < 외부 간격.
- 간격만으로 그룹핑이 표현되어야 함 — 선이나 배경색 없이도.

## 게슈탈트 원칙

### 근접성 (Proximity)
가까운 요소는 같은 그룹으로 인식. CRAP의 Proximity와 동일 원리.

### 유사성 (Similarity)
같은 색/크기/형태의 요소는 같은 카테고리로 인식. 버튼은 버튼끼리, 레이블은 레이블끼리 일관되게.

### 폐합 (Closure)
불완전한 형태도 뇌가 완성함. 최소한의 라인으로 "카드"나 "영역"을 암시 가능. 4변 테두리 대신 상하 2변만으로도 영역 인식 가능.

### 연속성 (Continuity)
시선은 부드러운 경로를 따름. 요소 배치가 자연스러운 흐름을 만들어야.

### 전경/배경 (Figure-Ground)
중요한 건 "떠 보여야" 하고, 배경은 "물러나야". 팝업 = 전경, 딤 처리된 뒤 = 배경.

## 시각적 위계 (Visual Hierarchy)

### 크기와 무게로 위계 표현
- 중요한 것 = 크게, 굵게
- 덜 중요한 것 = 작게, 가볍게
- 타이포 스케일 비율: Major Third(1.25) 또는 Perfect Fourth(1.333)
- 예: base 16px → 20 → 25 → 32 → 40

### 색상과 대비로 위계 표현
- 주의를 끌어야 할 요소 = 높은 대비 (액센트 색상, 밝은 텍스트)
- 보조 정보 = 낮은 대비 (회색 텍스트, 낮은 알파)
- CTA 버튼은 화면에서 가장 대비가 강한 요소여야 함

### 위치와 시선 흐름
- Z패턴: 좌상 → 우상 → 좌하 → 우하. CTA는 Z의 끝점(우하)에 배치.
- F패턴: 텍스트 중심 레이아웃. 좌측 정렬 + 상단에 핵심 정보.
- 바텀바에서: 주요 액션은 우측(Z패턴 끝), 보조 액션은 좌측.

### 시각적 무게와 균형
- 큰 요소, 어두운 요소, 채도 높은 요소 = 무거움
- 작은 요소, 밝은 요소, 채도 낮은 요소 = 가벼움
- 비대칭 균형: 한쪽에 큰 요소, 반대쪽에 여러 작은 요소로 균형
- 의도적 여백(negative space)은 시선을 집중시키는 도구

## 비례와 조화 (Proportion)

### 요소 크기 대비 장식 비율
- 테두리 두께는 요소 크기에 비례해야 함
- 400px 카드에 25px 테두리 = 과함 (6.25%). 1-2px이 적절 (0.25-0.5%)
- 작은 버튼(56px)에 1px 테두리 = 적절. 같은 1px이라도 큰 패널(500px)에서는 너무 얇을 수 있음
- 규칙: 장식 요소의 시각적 무게가 콘텐츠를 압도하면 안 됨

### 패딩 비례
- 작은 컴포넌트(버튼): 패딩 8-12px
- 중간 컴포넌트(카드): 패딩 16-24px
- 큰 컴포넌트(섹션): 패딩 24-48px
- 패딩은 콘텐츠에 "숨 쉴 공간"을 주되, 요소를 "흩어 보이게" 만들면 안 됨
